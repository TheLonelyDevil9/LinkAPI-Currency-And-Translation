## LinkAPI Userscript Notes

- Always bump the userscript metadata `@version` when changing `LinkAPI USD And English.user.js`; Violentmonkey will report "No update found" if the raw script version does not increase.
- After pushing a userscript fix, verify the raw GitHub URL serves the new `@version` before telling the user to update.
- Current scope (v6.0): CNY-to-target-currency conversion for text and split-node amounts (default target from detected location), live FX rates, automatic VAT for detected EU/UK locations on payable amounts, a floating currency-settings widget, the account-menu `Show USD values` toggle, redemption-row compaction, conservative route-scoped settings persistence, stale-artifact cleanup, and models.dev price comparison on pricing/model surfaces. Nothing else is active.
- Live FX uses frankfurter.dev v1 `latest?base=CNY` (all symbols), whose response shape is `{rates:{...}}`. The v2 `/v2/rates` endpoint returns an array of `{quote, rate}` objects instead; do not switch endpoints without updating parsing. The cache key is `tld-linkapi-cny-usd:fx:all-rates:v2` with a 60s TTL, and any future automated checks must seed it fresh so conversion assertions stay deterministic at the 0.146201 fallback rate.
- Location detection uses `ipapi.co/json/` with a timezone-map fallback, cached 24h under `tld-linkapi-cny-usd:user-location`. It feeds only the default target currency and the VAT rate; the widget's manual currency choice persists in `tld-linkapi-cny-usd:target-currency`/`:target-country`.
- models.dev pricing notes must stay idempotent through `syncPriceNotes`. Unconditionally clearing and re-appending notes re-triggers the body MutationObserver every animation frame; only rewrite notes when content changed. Note tooltips must keep naming the matched models.dev `fullId` so wrong fuzzy matches are auditable.
- Keep `@grant none`: models.dev serves `Access-Control-Allow-Origin: *` and frankfurter.dev is CORS-enabled, so page-context `fetch` works. If LinkAPI ever adds a restrictive `connect-src` CSP, move these loads to `GM_xmlhttpRequest` with `@connect` metadata instead of dropping the features.
- Usage-log cost chips may split the currency mark and amount across nested text nodes, for example `¥` and `0.290984` in separate spans. Keep split-node currency conversion covered when touching the converter, including re-conversion after a fresh FX rate arrives.
- Keep the redemption-code input/button compaction when supporting old and new layouts. The new layout needs the redemption input capped around `560px` and the redeem button forced back to compact/auto width; rollback compatibility work once removed this helper and made the control stretch again.
- Persist page settings conservatively in localStorage: remember route-scoped filters, page controls, and toggles, but do not store API keys, tokens, prompts, messages, redemption codes, credentials, or textarea content.
- The script must not modify the LinkAPI `chats` localStorage key. The old CC Switch/Cherry Studio/FluentRead import normalization hook was removed in 4.1; keep `chats` untouched.
- Removed in the 4.x slim-down: Chinese-to-English translations, table sorting, log auto-refresh and the log helper cluster, `00:00` time shortcuts, dashboard token totals and their fetch/XHR hooks, the dashboard model filter, and the floating `USD + EN` pill (replaced by the account-menu toggle). Treat these removals as deliberate; read `lessons.md` before reintroducing any of them.
- When removing helper features, also clean stale DOM classes, storage keys, injected controls, and helper stylesheet rules through `removeStaleArtifacts`. The removed model filter once left `.tld-linkapi-cny-usd-hidden-by-model-filter` and `tld-linkapi-cny-usd:model-filter` behind, which could keep models such as GPT-5.5 hidden after updates.
- Before planning or editing, read `lessons.md` and apply its history-derived guardrails; do not repeat known project mistakes.
- Validate script changes with `node --check "LinkAPI USD And English.user.js"`, then load the userscript in Violentmonkey against a live LinkAPI page and confirm currency conversion, the account-menu toggle, and models.dev pricing notes behave before telling the user to update.
- After completing and validating repository changes, commit and push them to the configured GitHub remote unless the user explicitly says not to.
